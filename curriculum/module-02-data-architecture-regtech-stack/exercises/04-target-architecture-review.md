# Uebung 4: Target Architecture Review

> **Schwierigkeit:** Advanced
> **Dauer:** 2-3 Stunden

## Kontext

Zum Abschluss von `M2` soll aus den vorigen Artefakten ein kleines, reviewbares
Zielbild entstehen.

## Aufgabe

Erstelle eine Referenzarchitektur fuer einen kleinen RegTech-Stack, der
mindestens verbindet:

- zwei Quellsysteme
- ein kanonisches Modell
- zwei Kontrollpfade
- einen Review- oder Case-Pfad
- Audit oder Reporting

## Erwartete Arbeitsschritte

1. Zeichne die Komponenten und ihre Datenfluesse.
2. Definiere Domain Boundaries und Ownership.
3. Benenne die drei groessten Risiken oder Failure Modes.
4. Formuliere eine kurze Bruecke zu `M3` oder `M4`.

## Bewertungskriterien

- klare Verantwortlichkeiten
- plausible technische Priorisierung
- sichtbare Betriebs- und Audit-Perspektive
- brauchbarer Anschluss an spaetere Module
